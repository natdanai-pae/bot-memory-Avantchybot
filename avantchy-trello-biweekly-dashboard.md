# Trello Biweekly Dashboard Tracking

User plans to make this Trello work-tracking summary every two weeks.

## Standard Output Format

When asked to evaluate/follow up Trello work, use the dashboard format developed for the GAA board:

- Team potential/performance dashboard by person:
  - ศักยภาพเด่น
  - ผลงานสำคัญ
  - บทบาทที่เหมาะสม
  - ระดับความพร้อม
  - ข้อเสนอพัฒนา
- Risk dashboard:
  - ใครล่าช้า
  - งานค้าง
  - งานตกหล่น
  - งาน active ที่เลยกำหนด
  - งานไม่มี due date
  - งานไม่มี owner
  - งาน due soon / ต้องไล่ตามทันที

## Individual Evaluation Format

For each person, also prepare an individual summary with these headings:

- หัวข้องาน
- รายละเอียดเบื้องต้น
- ข้อเสนอแนะ
- ปัญหา
- แนวทางแก้ปัญหา
- ภาพรวม
- สรุปผลประเมิน

This should be suitable for a director reviewing work progress and coaching staff every two weeks.

## Interpretation Notes

- State clearly that the assessment is based on Trello board evidence, not formal HR evaluation.
- Use exact summary date.
- Define criteria explicitly:
  - ล่าช้า = active cards past due date
  - งานค้าง = To Do / Pending / Doing cards not closed
  - ตกหล่น = active cards without owner or due date
- Prefer visible infographic/image output for Telegram delivery.
